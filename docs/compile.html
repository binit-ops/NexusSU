Compilation Guide

This guide covers how to compile both the NexusSU Manager App and your patched Android kernel.

Part 1: Building the Manager App (APK)

The Manager App is built using Gradle. The build process automatically invokes the Android NDK to compile the native su binary and nexussu_daemon C sources directly into the application assets.

Method A: GitHub Actions (Recommended)

The easiest way to build the Manager App is via automated CI. The repository includes a pre-configured workflow.

  1. Push your commits to the main branch of your repository.
  2. Navigate to the Actions tab on GitHub.
  3. Select the Build Manager APK workflow and trigger it manually or via push.
  4. Once complete, download the compiled NexusSU-Manager-APK artifact from the run summary.

Method B: Local Build (Android Studio / CLI)

To build locally, ensure you have JDK 17+, Android SDK, and NDK (v25.1.8937393+) configured in your environment.

$ git clone https://github.com/your-username/NexusSU.git
$ cd NexusSU
$ ./gradlew assembleRelease

The output APK will be generated at: manager/build/outputs/apk/release/app-release.apk.

Part 2: Building the Kernel (Proton Clang)

Before compiling, ensure your kernel source tree is patched using the instructions in the Kernel Integration Guide. We recommend using Proton Clang for legacy 4.9–5.4 kernels to ensure maximum performance and modern LLVM optimizations.

1. Install Host Prerequisites

Ensure your build host has the required toolchain dependencies installed:

$ sudo apt update && sudo apt install -y build-essential bc bison flex libssl-dev libncurses-dev git

2. Build Commands

Execute the following commands to fetch the toolchain, set up the environment, and compile the kernel.

Terminal 💻 Kernel Cross-Compilation 

# 1. Clone Proton Clang (Depth 1 for speed)
git clone --depth=1 https://github.com/kdrag0n/proton-clang.git ~/proton-clang

# 2. Export Target Architecture & Environment Setup
export ARCH=arm64
export SUBARCH=arm64
export O=out

# 3. Export Proton Clang Paths & LLVM Tools
export PATH="$HOME/proton-clang/bin:$PATH"
export CC="clang"
export CROSS_COMPILE="aarch64-linux-gnu-"
export CROSS_COMPILE_ARM32="arm-linux-gnueabi-"
export AR="llvm-ar"
export NM="llvm-nm"
export OBJCOPY="llvm-objcopy"
export OBJDUMP="llvm-objdump"
export STRIP="llvm-strip"

# 4. Navigate to Patched Kernel Root & Create Build Dir
cd /path/to/your/kernel/source
mkdir -p $O

# 5. Generate Defconfig
make O=$O OEM_defconfig

# 6. Compile Kernel Image
make O=$O -j$(nproc)

Upon success, the kernel binary (typically Image.gz-dtb or Image) will be saved in out/arch/arm64/boot/. You can zip this image using AnyKernel3 for flashing via TWRP or fastboot.

Boot Daemon Setup: The Manager App drops the nexussu_daemon binary to /data/adb/nexussu/bin/ upon first run. For automatic boot execution, ensure your device ramdisk or init scripts trigger this binary on startup.
