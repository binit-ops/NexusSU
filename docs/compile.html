Building the Engine

Because legacy GKI 1.0 and pre-GKI kernels do not support universal loadable modules, the NexusSU hooks must be compiled directly into the kernel source tree. This guide covers cross-compiling your modified kernel entirely on an Android host device using a PRoot environment.

1. Prepare the Build Environment

Spin up an Ubuntu PRoot environment within Termux and install the required build dependencies.


# Update package lists and install core compilation tools
apt update && apt upgrade -y
apt install git make bc bison build-essential zip curl flex \
    libssl-dev libfl-dev wget gcc-aarch64-linux-gnu -y

2. Fetch the Toolchain & Source

Download your device's specific kernel source code and a modern toolchain like Proton-Clang or Eva GCC to handle the cross-compilation.


# Clone the Proton-Clang toolchain
git clone --depth=1 https://github.com/kdrag0n/proton-clang.git ~/toolchain

# Clone your device's specific kernel source (Replace URL with your OEM source)
git clone --depth=1 https://github.com/OEM/kernel_source.git ~/android-kernel
cd ~/android-kernel

# At this stage, inject the NexusSU C hooks into fs/exec.c, fs/readdir.c, etc.

3. Cross-Compile the Kernel

Export your environment variables, generate the defconfig for your specific device architecture, and initiate the build sequence. Be sure to replace vendor/your_device_defconfig with your actual config name.


# Export toolchain paths
export PATH="$HOME/toolchain/bin:$PATH"
export CROSS_COMPILE=aarch64-linux-gnu-
export CROSS_COMPILE_ARM32=arm-linux-gnueabi-
export CC=clang
export ARCH=arm64

# Clean the tree and generate the configuration
make clean && make mrproper
make vendor/your_device_defconfig O=out

# Start the compilation (Using 8 threads)
make -j8 O=out \
    CC=clang \
    CROSS_COMPILE=aarch64-linux-gnu- \
    CROSS_COMPILE_ARM32=arm-linux-gnueabi-

4. Packing the Boot Image

Once compilation succeeds, the raw kernel image will be output to out/arch/arm64/boot/Image.gz-dtb. You must repack this into a bootable image using tools like Magiskboot or Android Image Kitchen (AIK) before flashing it via Fastboot or a secondary Bugjaeger host.


# Example Magiskboot repack sequence
./magiskboot unpack stock_boot.img
cp out/arch/arm64/boot/Image.gz-dtb kernel
./magiskboot repack stock_boot.img nexussu_patched_boot.img
Warning: Always ensure you have a fallback method to restore your partitions (such as MTK Client for MediaTek devices) before flashing custom legacy kernels. A single misaligned pointer in your source injection will cause an immediate bootloop.
