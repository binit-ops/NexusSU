Legacy Source Injection

For pre-GKI kernels (4.14, 4.19, 5.4), NexusSU relies on direct source tree injection. By modifying the kernel's core execution and file system files before compilation, we achieve an embedded, undetectable root environment.

Step 1: The Core Header (nexussu.h)

This header acts as the central brain. It defines the privilege escalation function that swaps the current process credentials to UID 0 (Root). This file is placed in the include/linux/ directory of the kernel tree.


#ifndef _NEXUSSU_H
#define _NEXUSSU_H

#include <linux/cred.h>
#include <linux/uidgid.h>

/* Function to check if a UID is granted root via the Manager App */
extern bool nexussu_is_granted(kuid_t uid);

/* Core escalation function */
static inline void nexussu_escalate(void) {
    struct cred *new_creds;
    
    /* Prepare a new credential block */
    new_creds = prepare_creds();
    if (new_creds != NULL) {
        /* Assign Global Root IDs */
        new_creds->uid = GLOBAL_ROOT_UID;
        new_creds->gid = GLOBAL_ROOT_GID;
        new_creds->euid = GLOBAL_ROOT_UID;
        new_creds->egid = GLOBAL_ROOT_GID;
        
        /* Commit the elevated credentials back to the task */
        commit_creds(new_creds);
    }
}

#endif /* _NEXUSSU_H */

Step 2: Execution Interception (fs/exec.c)

To trigger the escalation, we inject our logic directly into the kernel's execution path. When an authorized app attempts to execute a command (like the su binary), the kernel intercepts it and elevates the credentials natively.

Maintainer Warning: When injecting blocks of custom logic directly into C files like fs/exec.c, it will inevitably cause line numbering shifts throughout the rest of the file. A character-by-character review is required to ensure you don't break existing kernel structures or misalign compiler macros. 

/* 
 * INJECTION TARGET: fs/exec.c
 * Locate the beginning of 'do_execveat_common' (or 'sys_execve')
 */

#include <linux/nexussu.h> /* Include our core header */

static int do_execveat_common(int fd, struct filename *filename,
                              struct user_arg_ptr argv,
                              struct user_arg_ptr envp,

{

    if (nexussu_is_granted(current_uid())) {
        nexussu_escalate();
        printk(KERN_INFO "NexusSU: Elevated execution for UID %d\n", current_uid().val);
    }
    /* --- NEXUSSU ESCALATION HOOK END --- */

    struct linux_binprm *bprm;
    int retval;
    /* ... */

Step 3: VFS Stealth (fs/readdir.c)

Granting root is only half the battle; hiding it is the rest. To bypass aggressive user-space anti-cheat systems, we intercept the Virtual File System's directory-reading capabilities. If a process attempts to list a directory containing our root binaries, we dynamically scrub the entries from the kernel's response buffer.


/* 
 * INJECTION TARGET: fs/readdir.c
 * Hooking the 'filldir' actor to hide specific files from user-space 
 */

#include <linux/nexussu.h>
#include <linux/string.h>

/* The name of the binary we want to hide */
#define HIDE_TARGET "su"

/* 
 * Our custom wrapper for the directory filler.
 * It checks the name of the file being read. If it matches our target,
 * it returns 0 (success) without actually writing the entry to the user's buffer.
 */
static int nexussu_filldir_hook(struct dir_context *ctx, const char *name, int namlen,
                                loff_t offset, u64 ino, unsigned int d_type)
{
    /* If the file is 'su', skip it completely */
    if (namlen == 2 && strncmp(name, HIDE_TARGET, 2) == 0) {
        printk(KERN_DEBUG "NexusSU: Hid '%s' from directory listing\n", HIDE_TARGET);
        return 0; 
    }

    /* Otherwise, proceed normally (you would call the original filldir here) */
    // return original_filldir(ctx, name, namlen, offset, ino, d_type);
    return 0;
}
