Professional Kernel Integration

NexusSU utilizes a zero-file, syscall-hooking architecture. It leaves no traces on the filesystem and uses a prctl magic number for kernel-to-userspace communication.

Method 1: Automated Script (Recommended)

Run the provided apply.sh script against your kernel source tree.

$ chmod +x apply.sh
$ ./apply.sh /path/to/your/kernel/source

Method 2: Manual Source Injection

If the automated script fails due to heavy ROM source modifications, you can manually patch the kernel using the following 12 steps.
